1. 背景：TP 与 DP 的正交性

在 Megatron 的设计中，Tensor Parallelism (TP) 和 Data Parallelism (DP) 是正交的：

核心原理：TP 只对 Linear / Embedding / LM-Head 等权重进行切割。在 TP 域中，未切分的权重（如 LayerNorm）在每个 rank 上是完整保存的，且由于它们看到的是相同的数据，backward 产生的梯度天然一致，无需 TP 内同步。

因此：

  • 切分的权重（Linear）：TP 通信（AllReduce/AllGather）保证各分片拿到正确梯度
  • 未切分的权重（Norm, bias）：各 rank 输入相同 → 梯度天然相同 → 无需同步
  • DP 不侵入 TP 域：DP 只在 TP group 之间做梯度同步
但是！当引入 Sequence Parallelism 后，这个「未切分权重看到相同数据」的前提被打破了。

2. TP 域的权重切分策略

层 权重 切分方式 Shape
ColumnParallelLinear
(QKV proj, FC1)		 weight 沿 dim=0 (输出维) 切分 [out/TP, in]
bias 沿输出维切分 [out/TP]
RowParallelLinear
(Out proj, FC2)		 weight 沿 dim=1 (输入维) 切分 [out, in/TP]
bias 不切分（完整复制） [out]
VocabParallelEmbedding weight 沿词表维切分 [vocab/TP, dim]
LayerNorm / RMSNorm weight, bias 不切分（完整复制） [hidden]
MoE Router weight 不切分（完整复制） [num_experts, hidden]

ColumnParallelLinear (列并行)

# Y = XA + b, 其中 A 沿列（输出维）切分: A = [A_1, ..., A_p]
# 每个 TP rank 计算: Y_i = X @ A_i
self.output_size_per_partition = divide(output_size, world_size)
self.weight = Parameter(torch.empty(
    self.output_size_per_partition, self.input_size  # [out/TP, in]
))
_initialize_affine_weight_gpu(self.weight, init_method, partition_dim=0)

RowParallelLinear (行并行)

# Y = XA + b, 其中 A 沿行（输入维）切分
# 输入也被切分: X = [X_1, ..., X_p]
# 每个 TP rank 计算: Y_i = X_i @ A_i, 然后 AllReduce 求和
self.input_size_per_partition = divide(input_size, world_size)
self.weight = Parameter(torch.empty(
    self.output_size, self.input_size_per_partition  # [out, in/TP]
))
_initialize_affine_weight_gpu(self.weight, init_method, partition_dim=1)

3. SP 引入的梯度同步问题

SP 的核心思想是：将 TP 中 Linear 层之间的冗余激活（每个 rank 都持有完整 [S, B, H]）改为各 rank 只持有 [S/TP, B, H]，从而节省显存。

纯 TP（无 SP）

LayerNorm 输入: [S, B, H]
每个 TP rank 看到完整序列
→ 梯度天然一致 ✅

TP + SP

LayerNorm 输入: [S/TP, B, H]
每个 TP rank 只看到部分序列
→ 梯度只是全局的 1/TP ⚠️

问题本质：

LayerNorm 的参数梯度 ∂L/∂γ = Σ_tokens(∂L/∂y_t · norm(x_t))

SP 下每个 rank 只累加了 S/TP 个 token 的梯度贡献，得到的是局部梯度，必须跨 TP group AllReduce SUM 才能得到完整的全局梯度。

受影响的参数

参数原因
LayerNorm.weight / LayerNorm.bias 逐 token 计算，SP 下每个 rank 只看到部分 token
RowParallelLinear.bias bias 作用在 ReduceScatter 后的分片输出上，同理只看到部分序列
Router.weight (MoE) Router 对分片后的 token 做路由决策
qk_layernorm 参数 对 Q/K 做 norm，输入也是分片的

而 Linear 层的 weight 不受影响！

关键洞察：Linear 层的 weight 梯度通过 forward/backward 中的 AllGather/ReduceScatter 通信操作已经隐式获得了完整序列的信息，不需要额外的梯度同步。
  • ColumnParallelLinear：前向 AllGather input 恢复为 [S, B, H] → backward 时 weight gradient 用完整 input 计算 → 天然正确
  • RowParallelLinear：前向 ReduceScatter output → backward AllGather grad_output → weight gradient 用完整 grad 计算 → 天然正确

4. Megatron 的 SP 梯度同步方案

4.1 标记参数：param.sequence_parallel = True

Megatron 中所有接收 SP 切分数据的"非 TP 切分"参数，在创建时都会被打上 sequence_parallel=True 标记：

# megatron/core/fusions/fused_layer_norm.py — Megatron 自带的 FusedLayerNorm
self.weight = Parameter(torch.empty(*hidden_size))
self.bias = Parameter(torch.empty(*hidden_size))
self.sequence_parallel = self.config.sequence_parallel
# set sequence parallelism flag on weight and bias parameters
setattr(self.weight, 'sequence_parallel', self.sequence_parallel)
setattr(self.bias, 'sequence_parallel', self.sequence_parallel)

# tensor_parallel/layers.py — RowParallelLinear.bias
setattr(self.bias, "sequence_parallel", self.sequence_parallel)

# transformer/moe/router.py — MoE Router weight
setattr(self.weight, 'sequence_parallel', self.config.sequence_parallel)

# transformer/moe/shared_experts.py — Shared Expert gate_weight
setattr(self.gate_weight, 'sequence_parallel', self.config.sequence_parallel)

4.1.1 不同 Norm 实现的 SP 梯度处理机制

Megatron 支持多种 Norm 实现，它们处理 SP 梯度同步的方式有所不同：

Norm 实现如何设置梯度同步方式备注
FusedLayerNorm
megatron/core/fusions/		 给 weight/bias 设 sequence_parallel=True finalize_model_grads → AllReduce SUM Megatron 原生实现，显式标记
TENorm (TE LayerNorm/RMSNorm)
transformer_engine.pytorch		 构造时传 sequence_parallel=True TE kernel backward 内部自动做 AllReduce SUM 不依赖 finalize_model_grads，kernel 级融合
nn.LayerNorm (PyTorch 原生) Megatron 的 torch_norm.py 中 assert not config.sequence_parallel 不支持 SP 如需使用，必须手动给参数设属性
关键发现：如果自定义模型使用 nn.LayerNorm 作为 Norm 层（而非 TENorm 或 FusedLayerNorm）， 必须手动给其 weight/bias 设置 sequence_parallel=True，否则 finalize_model_grads 不会收集该参数的梯度进行 AllReduce SUM，导致梯度不完整（仅为正确值的 1/TP）。

常见错误：误用 average_gradients_across_tp_domain=True（AVG）来代替 sequence_parallel=True（SUM）。 虽然在等长序列 + local mean loss 模式下"碰巧"能得到正确结果（local_mean × TP → AVG = global_mean）， 但在 per-token loss 模式下会导致梯度偏小 TP 倍。而且 AVG 和 SUM 的语义完全不同：
  • SUM：每个 rank 只有局部梯度，需要加起来得到完整梯度
  • AVG：每个 rank 已有完整梯度（仅有微小数值差异），取平均防漂移

4.1.2 TE Norm 的内部机制

# extensions/transformer_engine.py — TENorm 构造时传入 sequence_parallel
class TENorm:
    def __new__(cls, config, hidden_size, eps):
        norm_module = te.pytorch.LayerNorm  # 或 RMSNorm
        instance = norm_module(
            normalized_shape=hidden_size,
            eps=eps,
            sequence_parallel=config.sequence_parallel,  # ← 传给 TE
            zero_centered_gamma=config.layernorm_zero_centered_gamma,
        )
        return instance

# 当 sequence_parallel=True 时，TE LayerNorm 的 backward 会自动

4.2 训练步结束时统一 AllReduce

# finalize_model_grads.py
def _allreduce_non_tensor_model_parallel_grads(model, config, tp_group):
    """对 sequence_parallel 标记的参数梯度做 TP group AllReduce SUM"""
    if tp_group.size() <= 1:
        return

    grads_sum = []
    for name, param in model.named_parameters():
        if param.requires_grad:
            if config.sequence_parallel and getattr(param, "sequence_parallel", False):
                grads_sum.append(param.main_grad.data)

    if grads_sum:
        coalesced = _flatten_dense_tensors(grads_sum)
        torch.distributed.all_reduce(coalesced, op=ReduceOp.SUM, group=tp_group)

4.3 调用时序

Step 1：所有 microbatch 的 forward + backward 完成
Step 2：finalize_model_grads() 开始
→ DP gradient sync（AllReduce / ReduceScatter 跨 DP ranks）
Step 3：_allreduce_non_tensor_model_parallel_grads()
→ TP group AllReduce SUM for sequence_parallel params
Step 4：Embedding / Position Embedding grad sync（跨 PP stages）
Step 5：Per-token 梯度缩放（if enabled）：grad *= 1/global_num_tokens
Step 6：optimizer.step()

5. SP 模式下的完整数据流

输入: [S, B, H] (全量序列) │ ▼ VocabParallelEmbedding + ReduceScatter [S/TP, B, H] ← 进入 SP 表示 │ ╔══════════════════════════════════════════════════════════════╗ ║ Transformer Layer (重复 N 次) ║ ║ ║ ║ [S/TP, B, H] ║ ║ │ ║ ║ ├── LayerNorm ──→ [S/TP, B, H] (逐 token, 无通信) ║ ║ │ ║ ║ ├── ColumnParallelLinear (QKV) ║ ║ │ 前向: AllGather [S/TP]→[S] → matmul → [S, B, H/TP] ║ ║ │ 反向: ReduceScatter grad → [S/TP, B, H] ║ ║ │ ║ ║ ├── Self-Attention (在 H/TP 维度计算) ║ ║ │ ║ ║ ├── RowParallelLinear (Out proj) ║ ║ │ 前向: matmul → ReduceScatter → [S/TP, B, H] ║ ║ │ 反向: AllGather grad_output → [S, B, H] ║ ║ │ ║ ║ ├── Residual + LayerNorm → [S/TP, B, H] ║ ║ │ ║ ║ ├── ColumnParallelLinear (FC1) ║ ║ │ 前向: AllGather → matmul → [S, B, FFN/TP] ║ ║ │ ║ ║ ├── Activation (GeLU/SiLU) ║ ║ │ ║ ║ ├── RowParallelLinear (FC2) ║ ║ │ 前向: matmul → ReduceScatter → [S/TP, B, H] ║ ║ │ ║ ║ └── Residual → [S/TP, B, H] ║ ╚══════════════════════════════════════════════════════════════╝ │ ▼ AllGather 恢复全量序列 (for LM Head) [S, B, H] │ ▼ LM Head / Loss 计算 scalar loss
SP 的通信模式本质：
  • 纯 TP：RowParallelLinear 前向做 AllReduce（昂贵，且结果冗余）
  • TP+SP：将 AllReduce 拆为 ReduceScatter（前向）+ AllGather（在 ColumnParallel 前向时做）
  • 通信量相同，但中间状态从 [S, B, H] 降为 [S/TP, B, H]，显存减少 TP 倍

6. Loss 计算方式对梯度同步的影响

结论：不影响 AllReduce SUM 操作本身，但影响梯度的数值缩放语义。

传统模式

loss = sum(loss_per_token) / local_num_tokens

在 backward 前归一化

每个 rank 的梯度是局部平均

Per-Token Loss 模式 推荐

loss = sum(loss_per_token)

不归一化，最后统一缩放

每个 rank 的梯度是局部总和

传统模式的潜在问题

# 每个 rank 计算局部平均 loss
loss = torch.sum(loss_per_token * loss_mask) / loss_mask.sum()  # local mean
loss.backward()

# SP 参数的 AllReduce SUM 后:
# grad = local_mean_rank0 + local_mean_rank1 + ... + local_mean_rank_{TP-1}
# 如果各 rank 的 token 数相同 (= S/TP):
#   = TP * global_mean  (多了 TP 倍！但如果 DP 端已预期除以 TP 则 OK)
# 如果各 rank 的有效 token 数不同 (变长序列 / 不均匀 padding):
#   ≠ global_mean  (不精确！)

Per-Token Loss 模式（精确）

# forward: 只返回 loss_sum, 不归一化
loss_sum = torch.sum(loss_per_token * loss_mask)
num_tokens = loss_mask.sum()
loss_sum.backward()

# finalize_model_grads 最后:
global_num_tokens = all_reduce(num_tokens, group=dp_cp_group)  # 跨 DP+CP
for param in model.parameters():
    param.grad *= (1.0 / global_num_tokens)  # 统一缩放

# SP 参数的处理:
# 1. AllReduce SUM → 得到 ∂(global_loss_sum)/∂param (精确的全局 sum 梯度)
# 2. × (1/global_num_tokens) → 得到精确的全局平均梯度 ✅
Loss 模式SP 参数 TP AllReduce 后语义适用场景
sum / local_tokens → backward 各 local_mean 之和
等长序列 OK，变长不精确		 传统 GPT 预训练
sum → backward → /global_tokens global_sum 梯度 → /global_tokens = global_mean
精确		 Per-token loss（推荐）

7. average_gradients_across_tp_domain vs sequence_parallel

在 finalize_model_grads 中，average_gradients_across_tp_domain 的优先级比 sequence_parallel 高：

if getattr(param, "average_gradients_across_tp_domain", False):
    grads_avg.append(grad.data)          # → AllReduce AVG
elif config.sequence_parallel and getattr(param, "sequence_parallel", False):
    grads_sum.append(grad.data)          # → AllReduce SUM
属性场景ReduceOp原因
sequence_parallel Megatron 原生层 (Norm, bias, Router) SUM 各 rank 只看到部分序列，梯度是局部的 1/TP，需要求和
average_gradients_across_tp_domain HuggingFace 模型包装器等非 TP 切分的复制模块 AVG 各 rank 理论上梯度一致，AVG 是为了消除非确定性导致的漂移
为什么优先级更高？HF 模型不参与 SP 的序列切分，它看到的是完整数据。如果一个参数属于 HF 模块（即使同时被标记了 sequence_parallel），其梯度是在完整序列上计算的，不需要 SUM，只需要 AVG 防漂移。两种语义互斥。
属性冲突 Bug 警示：如果一个参数同时被设了 sequence_parallel=True 和 average_gradients_across_tp_domain=True， 由于 if...elif 结构，AVG 会覆盖 SUM，导致本应 SUM 的参数实际走了 AVG —— 梯度数值错误。

典型触发场景：自定义模型中，某个参数（如 adaLN.modulation）在模块 __init__ 中正确设了 sequence_parallel=True， 但后续代码又对该参数所属模块调用了批量标记函数将其覆盖为 average_gradients_across_tp_domain=True。 结果是正确的 SUM 被错误的 AVG 覆盖。

规则：对于同一个参数，两个属性不能同时设为 True。根据该参数的梯度语义选择其一。

使用场景

# models/huggingface/module.py — 自动对 HF 模块所有参数设置
class HuggingFaceModule(MegatronModule):
    def __setattr__(self, name, value):
        """
        Set average_gradients_across_tp_domain=True on all params.
        This keeps replicated weights synchronized and prevents drift
        due to non-determinism in HF models.
        """
        super().__setattr__(name, value)
        if isinstance(value, torch.nn.Module):
            for param in value.parameters(recurse=True):
                setattr(param, "average_gradients_across_tp_domain", True)

8. 总结

参数类型梯度如何正确何时同步
ColumnParallelLinear.weight
[out/TP, in]		 前向 AllGather input → backward 用完整 input 计算 wgrad
隐式正确		 无需额外 TP 同步
RowParallelLinear.weight
[out, in/TP]		 前向 ReduceScatter → backward AllGather grad_output → 用完整 grad 计算 wgrad
隐式正确		 无需额外 TP 同步
LayerNorm weight/bias
[hidden] 完整复制		 SP 下只看到 S/TP 个 token 的梯度
需要显式同步		 finalize_model_grads → TP AllReduce SUM
RowParallelLinear.bias
[out] 完整复制		 作用在 ReduceScatter 后的分片输出上
需要显式同步		 finalize_model_grads → TP AllReduce SUM
HF 模块参数
完整复制, 非 TP 切分		 各 rank 梯度理论相同，但存在非确定性
防漂移同步		 finalize_model_grads → TP AllReduce AVG
设计哲学：
  1. 能隐式解决的不显式做：Linear 的 weight 梯度通过 forward/backward 通信已隐式正确
  2. 标记 + 延迟同步：需要显式同步的参数在创建时打上 sequence_parallel 属性，统一在 finalize_model_grads 批量处理
  3. DP 不侵入 TP+SP 域：DP 只负责跨 TP group 的梯度同步，TP 域内的 SP 梯度修正由 Megatron 自己的 finalize_model_grads 处理
  4. Per-token loss 保证语义精确：先 SUM 再统一归一化，避免变长序列下的不精确
